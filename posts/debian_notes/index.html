ubuntu镜像相对较大，并且UI风格相对老旧，后面遇到了Debian，看中它的全社区维护、永久开源的承诺和更现代的UI风格，因此入了坑，在此爬坑留痕

账户权限设置

创建非root用户，并且设置ALL权限，否则不能使用sudo命令，想创建文件夹、安装docker等行为就无从谈起

1
2
3

su root
sudo adduser brew
sudo vim /etc/sudoers

添加上这行

1

brew ALL=(ALL:ALL) ALL

解决了国内docker被禁止之后，Linux下的docker安装部署问题，详情看这里：解决Docker国内网络问题

安装docker

1

sudo curl -fsSL https://gitee.com/tech-shrimp/docker_installer/releases/download/latest/linux.sh| bash -s docker --mirror Aliyun
1
2
3
4
5
6
7
8

# 参考自：https://www.runoob.com/docker/docker-compose.html
sudo curl -L "https://github.com/docker/compose/releases/download/v2.2.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

sudo chmod +x /usr/local/bin/docker-compose
sudo ln -s /usr/local/bin/docker-compose /usr/bin/docker-compose

# 验证docker-compose命令是否安装成功
docker-compose version

新增镜像站点

1

sudo vim /etc/docker/daemon.json

输入下列内容，最后按ESC，输入 :wq! 保存退出

1
2
3
4
5
6
7

{
    "registry-mirrors": [
        "https://docker.m.daocloud.io",
        "https://docker.1panel.live",
        "https://hub.rat.dev"
    ]
}

重启docker，让新增的镜像站点生效

1

sudo service docker restart

sudo docker exec -it <container_id> /bin/bash

# 运行容器实例
sudo docker start <container_id>

# 停止容器实例
sudo docker stop <container_id>

# 移除容器实例
sudo docker remove <container_id>
